# js检测手机是否安装某一app ------------------- 需求总是人来提及的。 如果在OC中检测手机中是否有某一个应用，这个还是可以判断的。 但是如果纯在一个[h5页面]()呢？或许，这在前端看来，似乎是不可能的事情。但是就像刚才所说的，如今大部分都是原生和h5混杂开发模式。 如果一个分享h5页面或者一个分享出来的宣传h5页面有这样跳转的需求呢？ 在我看来，好像也是不可能完成的。我一个页面怎么能检测手机呢？ ![alt title](../img/xinren.gif) #####[未雨绸缪]()一下。(万一以后有个美女产品妹子提这样的需求呢？你岂不是可以立马*满足*她) ------------------- ## 淘宝 ------------------- 难以解决的问题，我默默的打开了淘宝。 WAP页面。找啊找找啊找找啊找。虽然本人编码很菜，但是找代码的能力还是可以的。 来，看这里看这里。 ![alt title](../img/taobao.png) 虽然，他们把代码都压缩了，看起来很费劲。但我依然不想错失未来在妹子面前表现自己的机会。 然后，我就找到了。 ![alt title](../img/daima.png) 这个代码太难整理了，但是至少我明白了整体的思路。 于是，我就觉得，这个东西既然他们能做，那我也能做。 ------------------- ## js检测手机是否安装某一app(实现思路) ------------------- >[重点]()其实主要就是比较的是页面跳转到应用的这段时间(装逼的说法叫做时间阀值) >1，进入下载页面，如果手机中存在app，则跳转app,这个时候，网页进入后台从而js挂起，但这个期间大约还有600ms-1000msjs 仍然在执行。 >2，所以，我做了个定时函数，如果在当前页面所待的时间比这个阀值(跳转临界值)大的时候，则说明没有应用。这个时候，就可以进行下一步的跳转工作了。 ####参考代码 ``` var timeout, t = 1000, hasApp = true; //这个临界点比较关键 一般是在600-1000ms setTimeout(function () { if (hasApp) { alert('手机中存在该应用'); //实际上，这个时候已经跳转到应用中去了 } else { alert('没有应用'); //这里可以进行下一步跳转操作 } document.body.removeChild(ifr); }, 2000) var t1 = Date.now(); var ifr = document.createElement("iframe"); ifr.setAttribute('src', '#此处放置原生程序所留得入口地址#'); ifr.setAttribute('style', 'display:none'); document.body.appendChild(ifr); timeout = setTimeout(function () { var t2 = Date.now(); if (!t1 || t2 - t1 < t+100) { hasApp = false; } }, t); })(); ``` 好了，我该说的都说完了，改研究的也都研究了。就等机会了。
